# Novagram Android

Nicegram/Telegram Android fork with custom features, built from the GitHub repo `ferelking242/Novagram`.

## Run & Operate

```
./gradlew :TMessagesProj_App:assembleDebug          # Build debug APK (Google Play variant)
./gradlew :TMessagesProj_AppHuawei:assembleDebug    # Build Huawei variant
./gradlew :TMessagesProj_AppHockeyApp:assembleDebug # Build HockeyApp beta variant
```

Required env/files:
- `local.properties` — `sdk.dir`, `githubPackagesUser`, `githubPackagesToken`, `googleMapsApiKey`
- `gradle.properties` — `APP_PACKAGE`, `APP_VERSION_CODE`, `APP_VERSION_NAME`, `RELEASE_STORE_PASSWORD`, `RELEASE_KEY_ALIAS`, `RELEASE_KEY_PASSWORD`
- `TMessagesProj/config/release.keystore` — signing keystore (already present)
- Google Services JSON files are already present in each app module directory

## Stack

- **Language**: Kotlin + Java (Android), AGP 8.6.1, Gradle 8.10.1
- **DI**: Dagger Hilt 2.57.2 + KSP 2.1.0-1.0.29
- **Networking**: OkHttp 4.12.0, Retrofit 2.11.0
- **UI**: Compose (Kotlin plugin 2.1.0), compose-destinations 1.9.54, ViewBinding
- **Firebase**: BOM 32.7.0 (Crashlytics, Analytics, RemoteConfig, Messaging)
- **Build**: NDK 21.4.7075529, CMake 3.10.2 (for JNI layers)

## Where things live

```
TMessagesProj/            - Core Telegram/Nicegram library (Java + Kotlin, JNI/C++)
TMessagesProj_App/        - Google Play app wrapper module
TMessagesProj_AppHuawei/  - Huawei AppGallery wrapper with HMS billing
TMessagesProj_AppHockeyApp/ - HockeyApp/AppCenter beta variant
TMessagesProj_AppStandalone/ - Standalone/direct install variant
nicegram-features/        - Real Nicegram feature implementations (com.appvillis.nicegram.*)
novagram-stubs/           - Composite build: stub implementations for private Nicegram SDK modules
  └── assistant-core/     - All 150+ stub classes for com.appvillis.* packages
  └── features/*, core/*  - Thin modules that delegate to assistant-core
```

- Build config: `gradle.properties`, `local.properties`
- Version catalog: `novagram-stubs/gradle/libs.versions.toml` (aliased as `internalLibs`)
- SDK schema: `novagram-stubs/assistant-core/src/main/java/com/appvillis/`

## Architecture decisions

- **Composite stub build**: The private Nicegram SDK (`nicegram:assistant-core` etc.) is unavailable, so `novagram-stubs/` is an `includeBuild` that substitutes all `nicegram:*` module coordinates with stub implementations. This lets the project compile without the real SDK.
- **Single fat stub module**: All ~150 stub classes live in `assistant-core`; the 7 other sub-modules (`features/account-export`, `core/network`, etc.) simply depend on `:assistant-core`, satisfying all transitive dependency substitutions.
- **`internalLibs` version catalog**: The `settings.gradle` loads `novagram-stubs/gradle/libs.versions.toml` as the `internalLibs` catalog, providing consistent library versions used by `TMessagesProj/build.gradle` and `nicegram-features/build.gradle`.
- **TrustWallet Maven**: Root `build.gradle` loads `githubPackagesUser`/`githubPackagesToken` from `local.properties` to authenticate with `maven.pkg.github.com/trustwallet/wallet-core`.
- **`google-services.json`**: Each app variant module already contains its own `google-services.json`; no manual setup needed.

## Gotchas

- `local.properties` is git-ignored; must be recreated on fresh clones with `sdk.dir`, GitHub PAT credentials, and optional keys
- `gradle.properties` requires `APP_PACKAGE`, `APP_VERSION_CODE`, `APP_VERSION_NAME`, and keystore credentials
- The keystore password used is `nicegram` (both store + key); verify against actual `release.keystore` if signing fails
- NDK and CMake are required for the JNI build in `TMessagesProj`; Replit environment does not have them installed
- `com.appvillis.feature_nicegram_billing.R` is auto-generated by the `nicegram-billing` sub-stub module (namespace set to `com.appvillis.feature_nicegram_billing`)
- `nicegram-features/build.gradle` uses `kotlin-kapt` (deprecated) — may need migration to KSP if Kotlin 2.x strict mode is enabled
